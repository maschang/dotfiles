# Work style

- 進め方は「調査→計画→実装→セルフレビュー→変更点提示→ユーザーのGO」で進める。
- 実装に入る前に、差分範囲と方針を箇条書きで提示する。
- 破壊的変更・権限が強い操作・大量変更は先に止まって確認を取る。
- 過度に長文化せず、差分と根拠を明確に。
